O que é isso?

O banimento pela Meta geralmente ocorre por comportamento robótico e denúncias de SPAM. A Uazapi funciona como uma ponte inteligente. Ela não "anula" as regras da Meta (ninguém consegue), mas oferece ferramentas de segurança e comportamento humano para que seus envios não levantem bandeiras vermelhas nos servidores do WhatsApp.
Como funciona o controle de envios?

De acordo com o manual, a gestão desses envios em massa deve seguir estas diretrizes para minimizar riscos:

    Delay (Atraso entre mensagens): Você deve configurar um intervalo de tempo entre um disparo e outro. Enviar 20k mensagens de uma vez é um comportamento impossível para um humano.

    Uso de Instâncias: A API trabalha com instâncias. Para volumes de 20k, a recomendação implícita é a segmentação e o respeito aos limites de conexão da instância.

    Chatbot e Interações: O fato de algumas mensagens gerarem interação é excelente. A documentação prevê o uso de Chatbots e Gatilhos (Triggers). Quando o usuário responde, a API processa isso como uma conversa legítima, o que aumenta a "saúde" do seu número perante o WhatsApp.

1. Configuração Completa do Webhook (Eventos Essenciais)

Para um sistema de prefeitura, você não pode apenas enviar; você precisa saber se o cidadão recebeu e se ele respondeu. O Webhook deve ser configurado logo após a criação da instância.

O que é isso?
É o "ouvido" do seu sistema. Sem ele, você não saberá se a mensagem de massa foi entregue ou se o cidadão está pedindo ajuda.

Como funciona?
Você deve enviar um POST para o endpoint de configuração de webhook habilitando os eventos específicos de monitoramento.

Exemplo de Requisição (Para o Claude implementar):
HTTP

POST /webhook/set/{instance}
Content-Type: application/json
apikey: SUA_API_KEY

JSON de Configuração Recomendado:
JSON

{
  "webhook": {
    "enabled": true,
    "url": "https://seu-sistema-prefeitura.com.br/api/webhooks",
    "byEvents": false, 
    "events": [
      "messages.upsert",      // Recebe novas mensagens (Interação do cidadão)
      "messages.update",      // Status de entrega (Enviado, Entregue, Lido)
      "connection.update",    // Status da instância (Conectado/Desconectado)
      "qrcode.updated"        // Recebe novo QR Code se o anterior expirar
    ]
  }
}

    Dica de Ouro: Ao receber o evento messages.update, seu sistema pode marcar no banco de dados quais dos 20k envios foram realmente entregues, gerando relatórios de transparência para a prefeitura.

2. Verificação de Status e Gestão de Conexão

No seu modal de UI/UX, o usuário precisa de feedback visual (ex: "Aguardando leitura...", "Conectado com sucesso!").

Como funciona?
O Claude deve criar um "loop" de verificação ou usar o Webhook para atualizar o status no banco de dados. O endpoint principal é o de estado da conexão.

Exemplo de Requisição:

apikey: SUA_API_KEY

Exemplo de Resposta (Sucesso):

E se precisar trocar o número? (Logout/Desconexão):
Se a prefeitura decidir trocar o chip de uma das instâncias de massa, use o endpoint de logout para limpar a sessão antiga com segurança.

    Endpoint: DELETE /instance/logout/{instance}

    Dica de Ouro: Sempre execute o logout antes de deletar uma instância para evitar que a sessão fique "pendurada" nos servidores do WhatsApp.
